ADVANCED COMPRESSION DETECTION SOLUTION
========================================

PROBLEM STATEMENT:
- MediaPipe Pose cannot distinguish between rescuer's hands and victim's chest during contact
- Hands and chest merge into single region during compressions
- Simple hand tracking fails to detect vertical displacement accurately
- Need to measure compression depth and rate despite occlusion

SOLUTION IMPLEMENTED:
=====================

1. MOTION-BASED DETECTION (Primary Method)
------------------------------------------
Instead of trying to separate hands from chest visually, we track the MOTION of the entire chest region:

Key Insight: During CPR, the chest moves DOWN (compression) and UP (release) in a cyclical pattern.
We don't need to see the hands separately - we just need to track chest displacement.

Implementation:
- Track chest landmark position over time (Y-coordinate)
- Calculate velocity (rate of change in position)
- Detect compression cycles using peak/trough analysis:
  * Downward motion (velocity > 0.05) = Compression start
  * Upward motion (velocity < -0.05) = Compression release
- Measure vertical displacement between peak and trough

Advantages:
✓ Works even when hands completely occlude chest
✓ Doesn't require hand-chest separation
✓ Robust to skin tone variations
✓ Real-time performance

2. TEMPORAL FILTERING
----------------------
- Maintain 30-frame motion history (~1 second at 30fps)
- Use moving average to reduce noise
- Debounce detection (400ms minimum between compressions)
- Prevents false positives from camera shake or minor movements

3. VELOCITY ANALYSIS
--------------------
- Calculate frame-to-frame velocity of chest position
- Compare current velocity to recent average
- Detect significant changes indicating compression motion
- More reliable than absolute position tracking

4. QUALITY METRICS
------------------
Measures:
- Depth: Vertical displacement in normalized coordinates
- Rate: Compressions per minute (BPM)
- Quality assessment:
  * Good: 80-130% of ideal depth, 90-110% of ideal rate
  * Shallow: < 80% ideal depth
  * Deep: > 130% ideal depth
  * Slow: < 90% ideal rate (110 BPM)
  * Fast: > 110% ideal rate

5. MULTI-LANDMARK TRACKING
---------------------------
Tracks multiple body landmarks simultaneously:
- Chest center (calculated from shoulders and hips)
- Shoulder midpoint (reference for upper body)
- Wrist positions (for hand presence verification)

Uses shoulder position as stable reference point since shoulders don't move during compressions.
Calculates relative motion of chest compared to shoulders.

TECHNICAL ARCHITECTURE:
=======================

CompressionDetector Class:
- processFrame(): Main detection pipeline
- detectCompressionCycle(): Peak/trough detection using velocity
- calculateDisplacement(): Measures vertical movement
- calculateMetrics(): Computes depth, rate, and quality

Data Flow:
1. MediaPipe Pose → Landmarks (33 body points)
2. Calculate chest position from landmarks
3. Track motion history (position + velocity)
4. Analyze velocity patterns
5. Detect compression cycles
6. Calculate quality metrics
7. Display real-time feedback

ALTERNATIVE APPROACHES CONSIDERED:
===================================

A. Instance Segmentation (Not Implemented)
- Would require training custom model to segment hands vs chest
- Computationally expensive
- Requires large labeled dataset
- Overkill for this use case

B. Depth Mapping (Not Implemented)
- Requires depth camera or stereo vision
- Not available on standard mobile devices
- Complex calibration needed

C. Optical Flow (Prepared but not primary)
- Tracks pixel-level motion between frames
- Can detect motion in chest region
- Included as OpticalFlowDetector class for future enhancement
- More computationally intensive than landmark tracking

WHY THIS SOLUTION WORKS:
=========================

1. Physics-Based Approach:
   - CPR compressions create predictable vertical motion
   - Motion pattern is consistent regardless of visual appearance
   - Velocity changes are more reliable than position alone

2. Doesn't Require Separation:
   - Accepts that hands and chest merge visually
   - Tracks the combined region's motion
   - Motion of combined region = motion of chest (since hands follow chest)

3. Robust to Occlusion:
   - MediaPipe Pose tracks chest even when partially occluded
   - Uses multiple landmarks for redundancy
   - Temporal filtering reduces impact of brief tracking losses

4. Real-Time Performance:
   - Lightweight calculations (no deep learning inference)
   - Runs at 30fps on mobile devices
   - Immediate feedback to rescuer

IMPROVEMENTS OVER SIMPLE HAND TRACKING:

USAGE:
======

The system automatically:
1. Detects when hands are placed on chest
2. Starts tracking compression motion
3. Counts each compression cycle
4. Measures depth and rate
5. Provides real-time feedback:
   - "✓ GOOD" - Compressions are correct
   - "⚠️ PRESS DEEPER" - Insufficient depth
   - "⚠️ TOO DEEP" - Excessive depth
   - "⚠️ SPEED UP" - Rate too slow
   - "⚠️ SLOW DOWN" - Rate too fast

Display shows:
- Current depth (as percentage)
- Current rate (BPM)
- Quality assessment
- Compression count

CONCLUSION:
===========

This solution solves the hand-chest separation problem by:
- Not trying to separate them visually
- Tracking motion instead of appearance
- Using physics-based cycle detection
- Providing actionable real-time feedback

The key insight is that we don't need to see the hands separately - we just need to measure the motion they cause in the chest region.
